# AI PROJECT RULES

## Purpose
This document is the single source of truth for the entire project. Every page, component, API, database operation, and design decision must follow these rules.

---

## General Rules

- Read this file before starting any task.
- Follow these rules for every change.
- Maintain consistency throughout the project.
- Do not change the project architecture without permission.
- Do not change the selected technology stack.
- Do not install unnecessary libraries.
- Write production-ready code only.

---

## Technology Rules

Always use:

- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Lucide React
- Framer Motion
- Supabase
- Zustand
- React Hook Form
- Zod
- npm

Never replace these technologies unless instructed.

---

## Backend Rules

Use:

- Next.js API Routes
- Next.js Server Actions
- Supabase PostgreSQL
- Supabase Authentication
- Supabase Storage

Do not create a separate backend server.

---

## Forbidden Technologies

Never use:

- Firebase
- MongoDB
- Express.js
- Clerk
- Auth0
- Redux
- Bootstrap
- Material UI
- jQuery

---

## Code Rules

- Use TypeScript everywhere.
- Write reusable components.
- Avoid duplicate code.
- Keep components small and modular.
- Use meaningful variable names.
- Use environment variables for secrets.
- Keep the code clean and readable.

---

## Design Rules

The design must be:

- Premium
- Modern
- Minimal
- Elegant
- Professional
- Responsive

Inspired by:

- Apple
- Stripe
- Linear
- Shopify Horizon

Avoid generic template designs.

## UI Rules

- Use Manrope for headings.
- Use Inter for body text.
- Use an 8px spacing system.
- Use rounded corners.
- Use soft shadows.
- Use smooth animations.
- Use consistent spacing.

---

## Performance Rules

- Optimize images.
- Lazy load where appropriate.
- Keep components reusable.
- Avoid unnecessary re-renders.
- Write optimized code.

## Responsive Rules

Every page must work perfectly on:

- Desktop
- Laptop
- Tablet
- Mobile

## SEO Rules

- Use semantic HTML.
- Optimize metadata.
- Use accessible components.
- Keep performance high.

## Deployment Rules

- Supabase
